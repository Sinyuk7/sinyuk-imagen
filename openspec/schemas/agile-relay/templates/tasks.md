# 执行任务清单 (Tasks)

> 规则：
> - 必须使用标准 Checkbox：`- [ ]` / `- [x]`
> - 每个任务必须是**可执行 + 可验证**的最小单元
> - 按依赖顺序排列（先主链路，后清理）
> - 若任务涉及多个模块或无法在单个 session 内完成，应拆分为子任务（防止卡死）

---

## 1. 基础骨架与边界建立 (Scaffolding)
- [ ] 1.1 创建目标目录与模块文件
- [ ] 1.2 定义最小公开 contracts（类型 / 接口 / Dataclass）
- [ ] 1.3 建立私有实现边界（如 `_internal/`）

---

## 2. 核心实现 (Core Implementation)
- [ ] 2.1 实现或迁移主入口逻辑
- [ ] 2.2 迁移内部实现到私有模块
- [ ] 2.3 补充入口函数 INTENT / INPUT / OUTPUT 契约
- [ ] 2.4 对齐核心数据流与返回结构

---

## 3. 集成与调用方更新 (Integration)
- [ ] 3.1 更新调用方指向新入口
- [ ] 3.2 处理兼容层或删除旧逻辑
- [ ] 3.3 检查跨模块引用是否违规

---

## 4. 清理与收口 (Cleanup)
- [ ] 4.1 删除废弃模块或重复逻辑
- [ ] 4.2 清理无效导出 / 过时代码
- [ ] 4.3 更新必要文档或说明

---

## 5. 自检与验证 (Verification)

> 必须基于真实执行结果填写状态  
> 未执行项必须标记为 Not Run，禁止推测为 Pass

- [ ] 5.1 运行 `python3 -m compileall` 并修复错误
- [ ] 5.2 运行 `mypy` 并修复 error 级问题
- [ ] 5.3 执行 import 自检（无跨 Feature 私有依赖）
- [ ] 5.4 执行最小行为验证 / smoke test
